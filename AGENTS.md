# 代码规范

- windows 系统
- 使用 uv 管理 python 环境
- 避免使用过时的包，使用 `uv add <package>`，如非必要不要指定版本。
- 使用 ruff 格式化代码
- **类型检查：**Standard Mode。函数参数、返回值、类成员、复杂对象尽可能显式标注类型，无法标注的（如依赖包未导出该类型）标注为`Any`。使用 `# type: ignore` 忽略无法避免的类型警告。
- **注释：**函数和类必须使用注释说明功能、参数和返回值。复杂函数分块添加注释。处理 `torch.Tensor` 或 `np.ndarry` 时，如果有明确形状，必须注明。
- **日志：**添加必要的日志用于 debug，如模块的启动和终止。不要在短时间内反复执行的代码快内添加日志，不要添加无意义日志，避免刷屏。
